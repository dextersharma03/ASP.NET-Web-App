Assignment Project and Requirements

The following duties had to be completed:

Part 1: Develop a web application that keeps track of user's hits to the site, their IP, and current time zone.
Part 2: Develop a slideshow web app with shuffle, play, stop, next, and prev controls.
Part 3: Develop a computer store web app for user login, product purchase, and configuration.
Part 4: Upgrade the computer store to use a database for data management.


Part 1: Cookie Hit Counter

Original Description: Create a web app that keeps track of user's hits to the site, their IP, and current time zone.

Interpretation: This part was quite straightforward and implemented the same way.

Implementation: I made use of ASP.NET's cookies API and their API for obtaining IP Addresses. The local time was done on the javascript side.

Guide: Click refresh to update the counter. Click Reset to set the counter back to 0

Part 1!

Part 2: Slide Show

Original Description: Create a web app that implements a slideshow (of about 20 pictures). Allowing the user to shuffle, play, stop, click next, or prev.

Interpretation: This part was easier to implement than the assignment 2 version! The instructions were straightforward and implemented as instructed

Implementation: I made use the update panel to partially refresh the image from the database. The slideshow automatically starts and uses a timer set to 500ms for each image

Guide: Click any button to control the slide show.

Part 2!

Part 3: Computer Store (First Implementation)

Original Description: Create a web app similar to a computer store, where users can login, create, configure, and buy computer parts and entire computers.

Interpretation: This version of the store is hardcoded and does not use a database. This is quite challenging due to the data volume and cookies were used extensively.

Implementation: I utilized many cookies and helper classes (Constants.css) to manage data. All images were sourced from iconsfinder.com.

Guide: Navigate to Products > Computers to view available laptops. Configure and add them to your cart. Products > Computer Parts lists parts that can be added to your cart. Use the contact tab to view contacts. The feedback form validates input before submission. The login/create user option is accessible via "Log In" or your username. This page validates login/sign up and enables logout.

Part 3!

Part 4: Final Computer Store

Original Description: Create a web app implementing a database-driven computer store (upgrade from Part 3)

Interpretation: This was easier to implement than Part 3. The store now includes password recovery and a user-friendly cart with database integration.

Implementation: My database, defined in tableSetup.sql, facilitates product, order, and customer management. The implementation largely resembles Part 3 but with database integration. Dynamic creation of controls in C# server code was challenging, yet manageable with practice.

Guide: Similar to Part 3. New features include password recovery (accessed on login page). The cart is improved, allowing item quantity updates and automatic cart/database synchronization.

Part 4!
